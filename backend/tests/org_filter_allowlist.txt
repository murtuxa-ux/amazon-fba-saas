# Allowlist for backend/tests/test_query_lint.py.
#
# Format: one entry per line, "<repo-relative-path>:<lineno>" followed by
# a "# reason: <justification>" comment ON THE SAME LINE.
#
# Entries without a `# reason:` comment fail the lint test outright,
# forcing every exception to be documented at the point it's allowed.
#
# This file documents 30 db.query(<canonical>) call sites that the AST
# lint can't statically prove are tenant-scoped. They split into two
# categories:
#
#   (a) Pre-auth / cross-org-by-design User lookups: login, signup,
#       password reset, get_current_user, JWT-decoded WS handshake.
#       These intentionally have no app.current_org_id set yet — the
#       org context is being established BY this query (login) or
#       follows from globally-unique constraints (User.username/email
#       are UNIQUE; User.id from a signed JWT is authoritative).
#
#   (b) Builder-pattern false positives: the lint walks a single
#       statement's call chain. When code uses
#         q = db.query(M).filter(M.org_id == ...)
#         q = q.filter(...)
#         return q.all()
#       the .all() is on the variable, not on the original query
#       expression, so the lint can't follow the filter back to the
#       query. Per-site verification confirmed the org_id filter does
#       exist in the actual code path.
#
# Cleaning up category (b) would be straightforward (rewrite as a
# single expression chain, or invert the lint to be flow-aware) but
# is explicitly out of scope for PR C-2 — the multi-tenancy foundation
# ships first; query-style cleanups are a P1 follow-up.
#
# Tracked as P1 GitHub issue: "Query lint flow analysis + cleanup
# of pre-auth User queries" (opened alongside PR C-2).

# These run BEFORE Depends(tenant_session), so app.current_org_id is not

# discovering or validating the very identity that tenant_session keys on.
# In all cases the query keys on User.username, User.email, or User.id
# (all UNIQUE constraints) — a leak across orgs is not possible given
# those uniqueness invariants in the table schema.
